What is it?

Active Bean Fitness is a web-based application used to keep track of personal fitness activities. TODO

Goals

We're building an app about something we are passionate about. At the same time, we want to learn how to use many different modern web technologies. The application may incorporate many duplicate functions, such as multiple databases or multiple UI frameworks, for the sake of learning.

We want to build an application with millions of records that span multiple servers. We want to have so much data that efficient storage and retrieval become problems we must solve. The system should be a massive, distributed, fault-tolerant design.

Roadmap

The roadmap is a work in progress.

v1.0

  • AngularJS
  • HTML 5
  • Java 8
  • Spring Boot
  • Hibernate 4
  • MySQL
  • Elasticsearch on 1 server, 1 million records. Fallback to MySQL.
  • Full-text search on all records
  • User sign-up and authentication

v2.0

  • Elasticsearch on 2 servers, 10 million records
  • User ability to create private data

v3.0

  • Elasticsearch on 5 servers, 500 million records
  • In-memory database for warm data

v4.0

  • MongoDB with 1 million records
  • Cassandra with 1 million records
  • Additional security measures such as two-factor authentication

v5.0

  • Dart for middle-tier
  • PolymerDart for UI
