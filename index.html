Active Bean Fitness - Overview

What is the app?

This is a service used to record fitness activities, track progress, track equipment wear. TODO

Goals

We're building an app about something we are passionate about. At the same time, we want to learn how to use many different modern web technologies. The application may incorporate many duplicate functions, such as multiple databases or multiple UI frameworks, for the sake of learning.

We want to build an application with millions of records that span multiple servers. We want to have so much data that efficient storage and retrieval become problems that we must solve. The system should be massive, distributed, secure, performant, and fault-tolerant.

Roadmap

The roadmap is a work in progress.

v1.0

  • AngularJS -- done
  • HTML 5 -- done
  • Java 8 -- done
  • Spring Boot -- done
  • Hibernate 4 -- done
  • MySQL -- done
  • Elasticsearch on 1 server, 1 million records. -- done, but needs to be deployed
  • Full-text search on all records -- done
  • User sign-up and authentication -- done
  • App security (user log-in, token authentication, CSRF & XSS protection, etc.) -- done, except for CSRF token integration

v2.0

  • Elasticsearch on 2 servers, 10 million records
  • User session information stored in external data store (Redis?)
  • Establish user roles for viewing private/public data. For example, when a user logs in, they see their own fitness data and are unable to see other users' data.
  • iOS app

v3.0

  • Elasticsearch on 5 servers, 500 million records
  • In-memory database for data that is accessed most often
  • Android app

v4.0

  • MongoDB with 1 million records
  • Cassandra with 1 million records
  • Support two-factor authentication

v5.0

  • Support SSO through Google (and Facebook? and Twitter?)
